# Vol.0 起源与进化：从零到完全体的技术栈大阅兵！

> **摘要**：在这个博客框架满天飞的时代，小风酱不仅决定重复造轮子，甚至还给这个轮子装上了 V8 引擎和自动驾驶系统！  
> 本文将带你游览风风博客的底层架构，从前端的 TypeScript 防线，到后端的 Flask 魔法，再到为了极致偷懒而引入的企业级测试基建。

---

## 1. 动机...已经不知道动机是什么了！

老实说，社区上成熟的博客系统多如牛毛，`Wordpress`、`Hexo`、`Halo`……随便拎一个出来都能打。

但作为一个爱折腾的开发者，总觉得别人的轮子虽然圆，但不够顺手。其实最开始，这个博客只是一个想把 CSS 写出花来的纯前端静态页面，连文章都是手写 JSON 存着的。

但人生就是这么充满意外！为了能在网页上直接写文章、为了管理越来越复杂的画廊、为了不每次都手动改东西……这个原本的草台班子一路狂奔，莫名其妙地进化成了现在这个拥有前后端分离、JWT 鉴权、甚至配备了端到端测试的**完全体 CMS 系统**。

目标也从最初的好看就行，进化成了现在的三大法则：

1. **颜值第一**：必须要有毛玻璃、反光、粒子特效等精致的设计。
2. **交互有趣**：点击要有反馈，文章要有打字机特效，甚至还得有个目录小桌宠陪伴。
3. **工程硬核**：底层必须是正经的全栈架构，代码要足够强壮，绝不手软！

---

## 2. 前端：颜值与秩序的守护者

作为一个全栈~~干啥啥不行、摸鱼第一名~~的开发者，小风酱在选型时的原则只有两个：**开发要爽**，**维护要省**。

### 🎨 Vue 3 + Vite + TypeScript：神圣三位一体

为什么选 **Vue 3**？对小风酱来说，Composition API 写起来和吃薯片一样爽，完全没有 React 的 Hooks 依赖数组地狱。配合 **Vite** 闪电般的热更新，哪怕是改一个像素的阴影，也能瞬间在浏览器里看到效果。

**但为什么要上 TypeScript？**
因为......**编写 JS 一时爽，维护重构火葬场。**  
`JavaScript` 就像是在裸奔的疯狂原始人。当写下 `article.title` 的时候，内心必然是忐忑的；而当用 `TypeScript` 时，微软大战代码会温柔地说：“放心，`article` 里一定有 `title` 属性，而且它是个 string。”  
这不仅是为了装逼，更是为了回头看代码时，不会指着屏幕骂：“这坨 `data` 到底是个啥？！”

### 📦 Pinia：告别 Props 地狱

随着博客的进化，组件层级越来越深，需要共享的状态和各种弹窗也越来越多。如果还用老办法一层层传 Props，代码早就变成意大利面条了OAo！而 `Pinia` 就像是一个全局的魔法背包，任何组件都能随时掏出需要的数据和方法，完全不受层级限制。

本博客的全局状态管理就是靠 `Pinia` 实现的。无论是用户登录、文章列表、画廊友链，还有弹窗的开关，都在这里统一管理。这样一来，组件之间就完全解耦了，维护起来也轻松多了！

## 3. 后端：轻量却全能的魔法心脏

为什么不用 Java/Spring Boot？太重了，启动一次够喝杯冰红茶-3-。
为什么不用 Node/Express？因为没学！OAO。

**Python + Flask** 对小风酱来说是完美的平衡。启动快，代码少，装饰器路由写起来极其优雅。看看我们精挑细选的后端武备库：

- **Flask & Werkzeug**：提供最基础的 Web 服务和路由响应。
- **Flask-SQLAlchemy**：ORM 兵工厂，不写原生 SQL 是最后的倔强！
- **Flask-JWT-Extended**：博客的绝对防线！它提供了 Access Token (门票) + Refresh Token (凭证) 的双重鉴权机制，配合前端拦截器，实现了无感刷新的丝滑登录体验。
- **Flask-Cors & python-dotenv**：解决跨域烦恼，以及安全地管理那些见不得人的环境变量密钥。

### 💾 数据库是...SQLite？！

“什么？2025 年了还用文件数据库？”
是的。对于一个个人博客来说，**SQLite 就是神**。

~~偷懒的理由~~ 选择 SQLite 的理由如下：

1. **无需配置**：不需要装 MySQL 服务，不需要管端口。
2. **备份方便**：直接复制 `instance/blog.db` 文件，备份完成。
3. **性能足够**：除非博客日活过万（做梦），否则 SQLite 的性能完全溢出。

## 4. 自动化基建：偷懒，才是第一生产力！

如果说上面的技术栈是一个常规博客的标配，那么接下来的东西，就是小风酱为了偷懒而点出的科技树awa。

在目录结构中，你会发现 `e2e` 文件夹和 `vitest.config.ts`、`playwright.config.ts`。  
**是的，这个个人博客居然上了企业级的自动化测试框架！**

为什么要搞这么夸张？  
因为小风酱发现，每次重构完代码，都要手动点开浏览器：  
_“登录行不行？文章能不能发？画廊图片加载没崩吧？友链能点吗？”_  
点完一圈，半个小时过去了。这简直是对生命力的极大浪费！QAQ

于是，**Vitest**（负责单元测试）和 **Playwright**（负责端到端 E2E 测试）闪亮登场！

- **Vitest**：像一个严厉的老师，秒级检查那些复杂的逻辑工具函数（比如日期格式化、排序算法）有没有算错。
- **Playwright**：像一个不知疲倦的无情点击机器。一键启动，它会自动打开一个真实的浏览器，模拟人类去输入密码、点击按钮、检查弹窗是否正常弹出。

这不仅极大地解放了双手，最关键的是——**当别人看到你在自动化测试时，那种不明觉厉的高级感，装逼效果简直满分啊！！！（战术后仰）**

## 5. 跨域与代理：Vite 的小小魔法

项目结构采用了标准的前后端分离模式。开发时，前端跑在 `localhost:5173`，后端跑在 `localhost:5000`。为了让前端能顺畅地呼叫后端，在 `vite.config.ts` 里做了一点小小的配置：

```typescript
// frontend/vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'http://127.0.0.1:5000', // 偷偷转发给 Flask 后端
      changeOrigin: true,
    },
    '/static': {
      target: 'http://127.0.0.1:5000', // 图片等静态资源也一并转发
      changeOrigin: true,
    }
  }
}
```
